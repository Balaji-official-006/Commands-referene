# Docker Commands for DevOps Engineers

This document lists commonly used Docker commands along with examples.
These commands are useful for DevOps engineers during development, deployment, and troubleshooting.

---

# 1. Check Docker Version

Check whether Docker is installed and view its version.

```bash
docker --version
```

Example output:

```
Docker version 24.x.x
```

---

# 2. Pull an Image from Docker Hub

Download an image from the Docker registry.

```bash
docker pull nginx
```

Use case: Download the Nginx image before running a container.

---

# 3. List Docker Images

Display all images available on the local machine.

```bash
docker images
```

Example output:

```
REPOSITORY   TAG       IMAGE ID
nginx        latest    abc123
```

---

# 4. Build a Docker Image

Create an image from a Dockerfile.

```bash
docker build -t my-nginx-app .
```

Options:

* `-t` → tag name for the image
* `.` → current directory containing Dockerfile

---

# 5. Run a Container

Start a container from an image.

```bash
docker run -d -p 80:80 nginx
```

Options:

* `-d` → run container in detached mode
* `-p` → port mapping (host:container)

Example: Access the container via port 80.

---

# 6. List Running Containers

Display currently running containers.

```bash
docker ps
```

Useful for verifying active containers.

---

# 7. List All Containers

Show all containers including stopped ones.

```bash
docker ps -a
```

---

# 8. Stop a Container

Stop a running container.

```bash
docker stop <container_id>
```

Example:

```bash
docker stop 8f23a7c9
```

---

# 9. Remove a Container

Delete a stopped container.

```bash
docker rm <container_id>
```

---

# 10. Remove a Docker Image

Delete an image from the local system.

```bash
docker rmi nginx
```

---

# 11. View Container Logs

Check logs generated by a container.

```bash
docker logs <container_id>
```

Example:

```bash
docker logs nginx-container
```

Used for troubleshooting application issues.

---

# 12. Execute Command Inside Container

Run a command inside a running container.

```bash
docker exec -it <container_id> /bin/bash
```

Options:

* `-i` → interactive mode
* `-t` → terminal

Used for debugging containers.

---

# 13. Inspect Container

View detailed configuration of a container.

```bash
docker inspect <container_id>
```

Displays:

* container network
* IP address
* volumes
* environment variables

---

# 14. Check Docker Disk Usage

View storage usage by Docker resources.

```bash
docker system df
```

Shows usage of:

* images
* containers
* volumes

---

# 15. Clean Unused Docker Resources

Remove unused containers, images, and networks.

```bash
docker system prune -a
```

Use with caution as it removes unused resources.

---

# Conclusion

Docker commands are essential for container management in DevOps workflows.
Common tasks include building images, running containers, checking logs, and troubleshooting services.

Maintaining command documentation helps DevOps engineers perform faster debugging and system management.
